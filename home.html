I am just getting started with GitHub.io, so do not expect much yet. This will be a place for discussions and for storing manuals for easy web access, I hope. Email me at ahlstromcj@gmail.com .

This site is navigable via the site tree at the left. This tree is patterned after Doxygen's navigation tree from the 2000s.

 

This projects on this site use the following technologies:

  • C. The waonc, midicvt, and the "Seq" projects use the C language to various degrees. This fact arises from them being derived from legacy code. The xpccutxx was written first in C as an exercise in creating a unit-test framework in a basic language. There are many things that can be done with simple functional code.
  • C++. The coding projects at this site use C++ heavily. It's our favorite language for it's ability to override operators, provide templates, a raft of standard algorithms, and extreme flexibility. Some balk at its sometimes verbose notation compared to Java, Python, etc., but it seems that every language eventually incorporates C++ constructs. (The reverse is true as well, with lambdas and closures.)
  • GNU Autotools. We got started with this long ago and simply find it usually easy to figure things out, even so far as creating M4 platform test macros. At some point we will experiment with cmake (CMakeList.txt!???) and meson In years of exploring other build systems, they all seem to try to replicate the stuff that autotools does, and end up being just as much a Frankenstein monster of sub-technologies.
  • ALSA. We got most of our ALSA experience by studying Seq24 code.
  • JACK. We got most of our JACK experience by studying the RtMIDI code, but have since gone beyond that in our vaguely-related fork.
  • Windows MM. Our radical fork of PortMIDI owes its genesis to Seq24, but luckily we had some prior experience with this API from work done a couple decades ago.
  • LaTex. Doing documentation in this markup language is powerful, expressive, flexible, and arcane. It is well supported and nearly bug-free. People write books with TeX/LaTeX. They have with Microsoft Word as well, but at a cost of a lot of labor and using a crummy editor.
  • Free42 skinning It was quite fun to make skins for Thomas Okken's Free42 calculator emulation, in the free42-resources project.
  • HTML/CSS/Javascript. Our initial experience in static HTML was obtained creating the eary web site of the Charleston Battery soccer team. Later work add CSS and Javascript to the mix. Work-work add ASPX experience which no way we use here.
 
 

As time goes on, this site will be improved.
